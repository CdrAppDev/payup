INTERNAL STRATEGY

Data Onboarding:
Crawl, Walk, Run

Without 835/837 data, nothing else works. This is our phased roadmap for getting hospital claims data onto the platform—starting today with what's possible, scaling toward full automation.

The Data Access Challenge

How claims data flows today

1 Hospital billing system generates 837 claim file
2 837 transmitted to clearinghouse (Availity, Change Healthcare, etc.)
3 Clearinghouse routes claim to payer
4 Payer adjudicates and sends 835 remittance back through clearinghouse
5 Hospital imports 835 into billing system, discards raw file

Why this is hard

  • ! Clearinghouses retain files for only 30 days in active mailboxes. Archive access (up to 6 months) requires restoration requests.
  • ! Most hospitals don't store raw EDI files. Data lives in billing system databases, not exportable files.
  • ! No major billing system offers self-service EDI export. All require IT involvement or vendor assistance.
  • ! 35%+ of small hospitals outsource billing. They may not have direct clearinghouse access at all.

Crawl: White-Glove Manual Onboarding

Start today with hospitals that have direct clearinghouse access and are motivated to participate. High-touch, low-scale—but proves the model and generates early data.

Initial Data Window

Focus on recent data available in active clearinghouse mailboxes. Historical analysis comes later.

⚠ Needs clearinghouse policy citations

Target Profile

  • In-house billing — Not outsourced to third-party RCM company
  • Direct clearinghouse access — Can log into Availity, Change Healthcare, or Office Ally
  • Motivated business office manager — Willing to spend 2-4 hours on onboarding
  • Acute financial pressure — Operating at loss, high denial rates, motivated to recover revenue

How It Works

1 Sign BAA — Required before any data sharing. Payup provides template.
2 Guided clearinghouse download — Payup CS schedules call, screen-shares, walks through portal download of 835 files.
3 Secure upload — Hospital uploads files via Payup secure portal or SFTP.
4 Contract rate ingestion — Hospital provides contracted rate schedules (often spreadsheets) for variance detection.
5 Recurring uploads — Schedule weekly or bi-weekly manual uploads until automation is in place.

What Payup Needs to Build

  • • Secure file upload portal (drag-and-drop, SFTP)
  • • Step-by-step guides for Availity, Change Healthcare, Office Ally, Optum
  • • Screen recording library for common clearinghouse workflows
  • • 835/837 X12 parser that handles format variations
  • • Contract rate table ingestion (Excel, CSV)
  • • BAA template and signing workflow
PHASE 2

Walk: Clearinghouse Authorization

Shift the technical burden from hospital staff to Payup. Hospitals authorize access; Payup pulls files directly from clearinghouse portals or receives automatic delivery.

~800-1,000
Addressable Hospitals

40-50% of small/rural hospitals can authorize clearinghouse access even if they can't export files themselves.

1-2 weeks
Setup Time

Hospital signs authorization. Payup configures clearinghouse connection. Automated flow begins.

Real-time
Data Freshness

Once configured, files flow automatically. No recurring manual work for hospital staff.

Two Authorization Models

Model A: Credential-Based Access

Hospital provides clearinghouse portal credentials. Payup logs in and downloads files on their behalf.

SecurityMedium — Shared credentials
Setup complexityLow
Hospital IT involvementNone

Model B: SFTP Delivery Configuration

Hospital (or Payup with authorization) configures clearinghouse to deliver 835 files to Payup's SFTP endpoint.

SecurityHigh — No credential sharing
Setup complexityMedium — Clearinghouse config required
Hospital IT involvementSometimes

What Payup Needs to Build

  • • SFTP infrastructure to receive inbound file deliveries
  • • Secure credential vault (if using Model A)
  • • Automated file retrieval system for major clearinghouse portals
  • • Authorization forms and data sharing agreements
  • • File monitoring and alerting (missing files, format errors)
  • • SOC 2 Type I certification (table stakes for authorization)
PHASE 3

Run: Direct Clearinghouse Partnerships

The endgame. Payup becomes an authorized data recipient within clearinghouse networks. Hospital authorization is a checkbox, not a configuration project.

2,100+
Full TAM Accessible

All small/rural hospitals using partner clearinghouses. Outsourced billing is no longer a barrier.

Minutes
Activation Time

Hospital checks a box in their clearinghouse portal. Data flows immediately.

Zero
Hospital Technical Effort

No file downloads, no SFTP configuration, no IT involvement.

How It Works

1 Payup establishes partnership with clearinghouse (Availity, Change Healthcare, etc.)
2 Payup appears as authorized app in clearinghouse marketplace/settings
3 Hospital grants authorization via single click in clearinghouse portal
4 Data flows via API — real-time or batch, depending on clearinghouse capabilities

Partnership Requirements

  • • SOC 2 Type II certification — Required by all major clearinghouses
  • • HITRUST certification — Increasingly required, especially by Change Healthcare
  • • Demonstrated market traction — Clearinghouses want partners with existing customer base
  • • Technical integration — Build to clearinghouse API specs, pass certification
  • • Revenue share or fees — Clearinghouses may charge per-transaction or annual fees

Timeline to Clearinghouse Partnership

Months 1-6
Build traction with Crawl/Walk phases. Get SOC 2 Type I.
Months 6-12
Pursue SOC 2 Type II. Begin clearinghouse partnership discussions.
Months 12-18
Complete HITRUST if required. Technical integration with first clearinghouse.
Months 18-24
Launch partnership. Expand to additional clearinghouses.

The Outsourced Billing Segment (~35% of CAHs)

For hospitals that outsource billing to companies like TruBridge, R1 RCM, or regional billing services, the hospital may not have direct clearinghouse access. This requires a different approach.

Option A: Hospital Requests Data from Billing Company

  • 1. Hospital instructs billing company to share 835/837 data with Payup
  • 2. Requires amendment to existing contract or data sharing addendum
  • 3. Billing company may charge fees for custom data exports
  • 4. Timeline: 2-6 weeks depending on billing company responsiveness
Friction level: High

Option B: Payup Partners with Billing Companies

  • 1. Payup establishes relationship with TruBridge, regional billing services
  • 2. Billing company offers Payup as value-add service to their clients
  • 3. Data flows through established billing company infrastructure
  • 4. Revenue share model with billing company
Friction level: Low (once partnership exists)

Risk: Billing Companies as Competitors

Some billing companies may view Payup as a competitive threat—especially if they offer (or plan to offer) similar underpayment recovery services. TruBridge, for example, has its own analytics and denial management offerings. Partnership discussions should probe for competitive concerns early.

Security & Compliance Requirements

835/837 files contain PHI. There are no shortcuts here—these requirements create a 3-4 month minimum runway before receiving hospital data.

Minimum Requirements (Day 1)

  • HIPAA policies documented
  • Risk assessment completed
  • Encryption (AES-256 at rest, TLS 1.2+ in transit)
  • Audit logging in place
  • Incident response plan
  • BAA template ready

The Bottom Line

File upload works for early adopters but won't scale. The long-term moat is clearinghouse integration—meeting hospitals where their data already lives. Crawl phase proves the model and builds traction needed for clearinghouse partnership conversations. Each phase unlocks the next.
